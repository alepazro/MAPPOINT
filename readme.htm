1.     Known Issues

Opening and converting previous version files

·         Files from earlier versions of MapPoint may not open properly. You should back up any files from earlier versions of MapPoint before opening them in MapPoint 2013.  Opening files from earlier versions converts the files to the MapPoint 2013 format and may change some of the existing formatting.  Converted files cannot be opened in the previous version of the software. Some formatting may change when opening older files in MapPoint 2013.

Setup

·         When installing MapPoint 2013, clearing the check box for Enable MapPoint Integration with Office will still install the feature, but it has to be enabled using each Microsoft Office applications specific COM Add-ins dialog box. For example, to add MapPoint actions to the Microsoft Word menus and toolbars, you need to open the Microsoft Word COM Add-ins dialog and select the MapPoint Office Add-in check box.

To modify this functionality through the Setup command line, use OFFICE_INTEGRATION as the parameter. Use 0 for off and 1 for on. For example, the following command line installs MapPoint 2013 without the Office integration feature:  msiexec /i <path>\data.msi /qr OFFICE_INTEGRATION=0 

·         If you install a previous version of MapPoint after installing MapPoint 2013, programs written using the MapPoint 2013 object model may no longer work properly. 

·         For more Setup issues, see the Troubleshooting Setup section later in this document.

Territories and data mapping

·         MapPoint includes various geographies such as State/Province and ZIP Code/Postcode to support data mapping and Territory functionality.  These geometries do not exist up to the same altitude for all countries. Consequently, it may be necessary to zoom in after completing a data map or while creating a territory that spans countries.

User Interface

·         There are known issues with floating objects, such as toolbars, not appearing on multiple monitors after the monitors are switched. For example, if you have multiple monitors, move a floating object from one monitor to another and then close the application, switch monitor positions in Control Panel, and start the application again, you will no longer be able to see the toolbar on either monitor. To work around this problem, restore the monitors to their original positions.

Programmability

·         When creating territories within the MapPoint Control, be sure to explicitly save map files by calling the Control's SaveMap, SaveMapAs, or CloseMap methods. If you close the Control or the application that hosts the Control without explicitly saving a map with territories by using one of these methods, all territory data may not be saved with the map. Another way to work around this problem is to hide the territory pane before closing the Control by replacing it with another pane, such as the Legend and Overview pane.  

·         When using the Microsoft Visual Basic® 6.0 Package and Deployment Wizard to create a custom setup program for MapPoint, you should use Visual Basic version 6.0 with Service Pack 4 or later. If you do not use this version of Visual Basic, your setup program will not run properly.

·         Do not use the trial version of MapPoint to design or deploy applications. 

·         The MapPoint ActiveX® Control is designed to work only in Visual Basic 6.0 and Microsoft Visual Studio® .NET applications that use Microsoft Windows® Forms. Using it in any other programming environment is not recommended.  

·         Installing MapPoint or the MapPoint embedded license only registers MapPoint for the current user. Before running a compiled application for a different user, run the following command line from the folder where MapPoint 2013 is installed: mappoint.exe /regserver 

·         The Map.Selection property is cleared if focus switches to a different application. In these cases, you should save the selection to a new variable the first time that you access it.

2.     Troubleshooting Setup

Because MapPoint 2013 installs system files that might be shared with other applications, close all applications (including antivirus software) before running Setup.

To start Setup for installing MapPoint 2013, insert the Setup disc in the DVD drive. If MapPoint does not run after running Setup and restarting Windows, the problem is most likely related to system files used by some antivirus applications. To resolve this problem, close your antivirus software and reinstall MapPoint.

If you need to reinstall MapPoint on Windows Vista or Windows 7, follow this procedure:  

1.       Turn off your antivirus software.

2.       Click Start, click Control Panel, click Programs, and then click Programs and Features.

3.       Select Microsoft MapPoint North America 2013, and then click Uninstall.

4.       Reinstall MapPoint from your DVD.

If you need to reinstall MapPoint on Windows XP, follow this procedure:

1.       Turn off your antivirus software.

2.       Click Start, and then click Control Panel.

3.       In Control Panel, double-click Add or Remove Programs.

4.       Select Microsoft MapPoint North America 2013, and then click Remove.

5.       Reinstall MapPoint from your DVD.

 

Install and run MapPoint over a network

To copy MapPoint 2013 Setup files to a server:

1.    Create an administrative installation of MapPoint 2013 on a network server. To do this, insert the MapPoint 2013 Setup DVD in the drive, then from the command prompt, open the Setup.exe program in administrative mode: x:\>Setup.exe /a (where x is the drive location).

2.    Follow the on-screen instructions to create a server image of the MapPoint 2013 Setup files on your network server.

3.    When you complete the administrative installation, all the required installation files for MapPoint 2013 will be on your server. Make sure the folder containing these files is shared to all the appropriate users and that they all have access to it.

To install MapPoint 2013 from a server:

1.    Now that you have created an administrative installation as noted earlier in this document, it is simple to install MapPoint from a client. From the client computer, use the Universal Naming Convention (UNC) path (that is, \\servername\foldername) to access the network share containing the Setup files for MapPoint 2013, and double-click Setup.exe.

2.    After Setup is complete, all the MapPoint 2013 program files are installed to the client computer. To start MapPoint, click Start, click All Programs, and then click Microsoft MapPoint North America 2013 or Microsoft MapPoint Europe 2013.

3.     Troubleshooting Printing Problems

If you encounter problems when printing from MapPoint 2013, such as no printout or partial printout, low print quality, or erratic text, there are several ways to correct the problem.

·         Check with the manufacturer of your printer to see if updated drivers are available. The manufacturer's Web site is usually a good place to check for updated drivers.

·         Changing your print quality may improve printouts from certain printers or in certain memory configurations. To change your print quality, go to the File menu and click Print. Under Options, select one of the options in the Map quality box. The print quality is still quite good in the lower resolution mode. Even if your printer supports high-resolution printing, you might choose to use the Draft option to speed up printing. You can also increase the margins so the map is smaller and prints faster.

If your printer displays a message or an error indicator, see your printer manual for information about how to correct the problem.

If you receive an "out of memory" error message when printing, check your printer settings in Control Panel. On the Start menu, click Control Panel, click Printers and Faxes, right-click the icon for your printer, and then click Properties. Verify that the amount of printer memory installed in the printer matches the setting specified in Control Panel. If this does not correct the problem, try printing in a lower resolution mode by changing the Map quality to Normal or Draft or try increasing your page margins or using portrait orientation so the map is smaller and prints faster.

If the legend symbols are unclear on your printout, look for a "dithering" setting in the Printer Properties dialog box. Setting dithering to None or Fine gives better results than Coarse.

If you receive a message that no printers are installed and you have a printer installed, make sure that one of the printers has been set as the default printer.

4.     Hardware Issues

If you use a Microsoft IntelliMouse® and are running the Windows XP operating system, the IntelliMouse wheel may not scroll correctly through directions. To correct this behavior:

1.       On the Start menu, click Control Panel, and then click Display.

2.       On the Appearance tab, click the Effects button.

3.       Clear the Use the following transition effect for menus and tooltips check box, and then click OK.

Some display drivers are known to cause problems with MapPoint. In these cases, check for updated display drivers from the hardware manufacturers Web site, and try changing the Display settings in Control Panel to different screen resolutions or number of colors.
